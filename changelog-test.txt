Changes in grsecurity-2.2.1-2.6.36.2-201012191316.patch:
---------------------------------------------------------------
Update to latest PaX release, includes x86 UDEREF rewrite that changes 
user/kernel accessors to accessing through the gs segment register.  This makes 
UDEREF incompatible with SSP, but closes the KERNEL_DS hole in a generic way, 
simplifies the UDEREF code, and makes KERNEL_DS only allow copies within the 
kernel (vanilla permits within the entire address space)
changes ACCESS_ONCE back to using const
fixed info-leak in hpsa scsi driver
harmless gcc warning fix in gracl.c
