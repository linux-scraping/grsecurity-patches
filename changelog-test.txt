Changes in grsecurity-2.2.1-2.6.36.2-201101010125.patch:
---------------------------------------------------------------
Two security fixes and two non-security fixes from Greg Beeley:
1) grsec ignored clock_settime in its time change logging and ignored the 
compat versions of time changing functions on 64bit kernels
2) grsec failed to enforce ptrace restrictions or perform auditing for the 
compat version of ptrace on 64bit kernels
3) add cap_valid() to capable_nolog()
4) standardize time change logging to always log prior to the system time being 
modified
5) found by me after further auditing: we missed the readdir/getdents/getdents 
compat syscalls in RBAC enforcement of hidden files
I've updated my build/test system to perform regression tests against the 
compat userland case so these problems don't occur again.

Changes in grsecurity-2.2.1-2.6.36.2-201012191316.patch:
---------------------------------------------------------------
Update to latest PaX release, includes x86 UDEREF rewrite that changes 
user/kernel accessors to accessing through the gs segment register.  This makes 
UDEREF incompatible with SSP, but closes the KERNEL_DS hole in a generic way, 
simplifies the UDEREF code, and makes KERNEL_DS only allow copies within the 
kernel (vanilla permits within the entire address space)
changes ACCESS_ONCE back to using const
fixed info-leak in hpsa scsi driver
harmless gcc warning fix in gracl.c
